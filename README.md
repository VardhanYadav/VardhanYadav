### 1. [Cognitive Drift and Hallucination Detection using GPT Models](https://github.com/VardhanYadav/Cognitive-Drift-and-Hallucination-Detection-using-GPT-Models)

![Python](https://img.shields.io/badge/Python-3776AB?style=flat&logo=python&logoColor=white)

*   Designed and implemented a comprehensive, model-agnostic evaluation framework to automatically detect hallucinations in GPT-based Large Language Models. Built a 6-component detection pipeline integrating- MPNet for semantic similarity measurement, DeBERTa-v3 for Natural Language Inference, TF-IDF for lexical grounding analysis, BM25 for document retrieval and context ranking, a Cross-Encoder for fine-grained relevance scoring, and a Knowledge Graph for structured fact verification. Each component targets a distinct dimension of hallucination, ensuring comprehensive coverage across semantic, logical, and factual aspects of model output.
*   The framework was evaluated on a large-scale dataset comprising 8,742 total samples, yielding 1,508 valid user-assistant interaction pairs drawn from five prominent LLMs — ChatGPT, Claude, Gemini, Perplexity, and Ultrachat. This multi-model evaluation enabled meaningful cross-architecture comparisons and surfaced clear performance hierarchies. Statistical validation using one-way ANOVA confirmed significant differences between models, with an F-statistic of 55.22 and p-value below 0.001, while Cohen's d effect size analysis quantified performance gaps ranging from small to large across consecutive model pairs.
*   The framework achieved an ultra-low hallucination detection rate of just 0.03%, alongside a citation accuracy of 95.65% and a combined Good or Excellent response quality rating of 91.7% across the full evaluation corpus. ChatGPT emerged as the top performer with a Hallucination Index of 26.09, driven by balanced scores across all pipeline components and the lowest deviation rate of 31.22%. Claude followed closely with a Hallucination Index of 27.67, offering reliable performance across multiple evaluation dimensions.
*   Component-level analysis revealed that Cross-Encoder scoring contributed the strongest relevance signal, while NLI verification — though the most challenging dimension for models to maintain — proved critical in identifying logical inconsistencies that semantic methods alone could not capture.

### 6. [Movie Recommendation System using Information Retrieval and Context Relevancy](https://github.com/VardhanYadav/Movie-Recommendation-System-Using-Information-Retrieval)

![Python](https://img.shields.io/badge/Python-3776AB?style=flat&logo=python&logoColor=white)
![HTML5](https://img.shields.io/badge/HTML5-E34F26?style=flat&logo=html5&logoColor=white)
![CSS3](https://img.shields.io/badge/CSS3-1572B6?style=flat&logo=css3&logoColor=white)
![JavaScript](https://img.shields.io/badge/JavaScript-HexColor?style=flat&logo=logoname&logoColor=white)

*   A content-based movie recommendation system that suggests 5 similar movies for any given film input, powered by natural language processing and information retrieval techniques. Built on the TMDB 5000 Movies & Credits dataset from Kaggle, covering thousands of films with rich metadata including genres, cast, crew, plot overviews, and keywords.
*   This project applies Information Retrieval (IR) principles to cinema — treating each movie as a document and computing pairwise similarity across a multi-field tag corpus (plot, genre, cast, keywords, director).
*   For a queried movie, retrieves its row from the similarity matrix, sorts all other movies by descending similarity score, and returns the top 5 results with percentage scores. Provides a shareable, browser-accessible UI with example inputs (The Dark Knight, Avatar, Inception, The Godfather, Interstellar) and live typeahead suggestions.
